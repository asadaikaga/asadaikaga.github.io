Configurating a new computer

上上周手里的ThinkPad X270不幸阵亡了。故障原因为主板type-C口短路。说来也怪我自己，洗主板之后没有仔细吹干，可能残留有水渍。一插上电就开始发烫，不该通的地方被水灌通了。 之前想体验12寸小屏电脑，购入了X270。当时装上了Arch Linux，尝试了一直想用的i3wm。自然而然地它变成了我的主力机，记笔记写论文都在用。 时至今日虽然短路问题被解决了，X270已复活；但当时主力机出了问题，我也确实吓了一大跳，又急得不行。 千辛万苦把用户目录（$HOME/）备份了，匆匆下单了新电脑，无机可用的我盯上了手里那台2015年的ThinkPad系列旗舰机，碳纤维外壳，搭载Intel 5代CPU的X1 Carbon Gen3。

2018年高中毕业的时候，我爹慷慨地让我继承他那本退休的单位电脑，也就是上文的那台X1 carbon。 其实它的弧线机身并不长在我的审美点上，相比来说更喜欢X270那种方方正正的。 但它当年给我带来了两大震撼：一是笔记本居然可以这么轻；二是笔记本休眠居然可以几乎不掉电（顺带一提，我现在也不知道为什么X1 carbon休眠掉电这么少，或许和它那块几乎赶上整机大小的巨型电池有关系）。 而如今它又给我带来了新的震撼：在15年给员工发8G内存256G固态旗舰机的单位，说实话也挺任性的（并没有自抬身价的意思）。

动手之前，也有犹豫。我知道这台X1 carbon的CPU要比X270早两代，总担心太重的系统它带不动。我爸妈寄过来的时候，它不幸地已经被装上了Win11。我和它相对无言，光标以龟速在滑行。 最后我也没敢给它上压力，选了Linux Mint 22.3 zena。这个发行版使用了Ubuntu 24.04.1，是大约两年前稳定版本，桌面环境也是Linux Mint官方的Cinnamon桌面，相比GNOME能节省不少计算资源。

又用回了Ubuntu。它在我手里还挺流畅的！（此处应该附图，如果我不偷懒的话，或许之后会更新）

Contents

这篇文章主要介绍我在一台新电脑上建立基于i3桌面、vim和latex的工作流的过程。 这次使用了Linux Mint zena（附带Cinnamon桌面）发行版作为案例。 将本文apt install命令替换为其他的安装命令，在其他发行版上也可以复现类似的工作流。 因此，使用其他发行版时，这篇文章仍有参考价值。

另外，虽然本工作流只使用i3桌面，但前半段安装过程中可以先登录Cinnamon桌面进行操作。

Second Update: 2026-07-17

本来文章讨论的是Linux Mint + i3 windows manager的设定。在ThinkPad X1 Carbon上运行的很不错的Mint Zena, 在新的这一台intel十代CPU的Let‘s note上就显得太轻量了。新一代的设定采用了Fedora 44的KDE Plasma桌面发行版，使用了Hyprland tiling windows manager, 我觉得体验仍然非常棒，配置还变简单了。如果有机会再写一篇文章介绍wayland的生态。

另外还有一些设定我觉得很有趣的，比如说nnn+trash，使用TUI进行文件管理；还有qutebrowser+mvp，可以用比较少的资源占用来观看YouTube视频。

Copy files from remote: tailscale, ssh, rsync, git

# install and setup tailscale
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up # manual login needed

# installing rsync
sudo apt install rsync

# install git
sudo apt install git

# ssh configurations
# connect without password
ssh-keygen -t rsa
# If you know the server you are connecting
# comment the next line out and replace with your user and domain
# ssh-copy-id user@domain

Browsing and Viewing: qutebrowser, redshift

# install browser
sudo apt install qutebrowser

# copy the configuration and bookmark to a new folder
mkdir -p "$HOME/.config/qutebrowser"
cp "$HOME/setup/dotfiles/qutebrowser/config.py" "$HOME/.config/qutebrowser/"
mkdir -p "$HOME/.config/qutebrowser/bookmarks"
cp "$HOME/setup/dotfiles/qutebrowser/bookmarks/urls" "$HOME/.config/qutebrowser/bookmarks"

# install redshift for temperature adjust
sudo apt install redshift

# if there is no redshift directory
dirname="$HOME/.config"
if [ ! -d "$dirname" ]
then
    echo "File doesn't exist. Creating now"
    mkdir -p -- $dirname
    echo "File created"
else
    echo "File exists"
fi
cp "$YOUR_PATH_TO_CONF/redshift.conf" "$HOME/.config/redshift.conf"

Windows Manager: i3conf (from now on you can use your i3 environment)

# If the distribution is ubuntu, use apt to install i3-wm
sudo apt install i3-wm

# move the dotfiles
mkdir -p "$HOME/.config/i3"
cp "$HOME/setup/dotfiles/i3/config" "$HOME/.config/i3/"

# polybar
sudo apt install polybar
dirname="$HOME/.config/polybar"
if [ ! -d $dirname ]
then
	mkdir -p -- $dirname
else
	echo "Folder exists"
fi
rsync -a "$HOME/setup/dotfiles/polybar/" $dirname

# i3-lock color
sudo apt intall i3lock-fancy

# feh for wall papers
sudo apt install feh

# picom
sudo apt install picom

# fcitx5 for input method (Chinese and Japanese)
sudo apt install fcitx5
sudo apt install fcitx5-mozc
sudo apt install fcitx5-chinese-addon

# brightness (which one is used for linux mint?)
sudo apt install brightnessctl

# alacritty (most important)
sudo apt install alacritty

dirname="$HOME/.config/alacritty"
if [ ! -d "$dirname" ]
then
	echo "Folder not exist, creating now"
	mkdir -p -- $dirname
	echo "Folder created"
else
	echo "Folder exists"
fi
cp "$HOME/setup/dotfiles/alacritty/alacritty.toml" "$dirname"
# rofi or dmenu
sudo apt install dmenu

The editor: vim, vimwiki, ultisnips, vimtex

# download vim
sudo apt install vim

# install git
sudo apt install git

# vim config files and Ultisnips config files
cp "$HOME/setup/dotfiles/vim/vimrc" "$HOME/.vimrc"
mkdir -p "$HOME/.vim/UltiSnips/"
rsync -av "$HOME/setup/dotfiles/vim/vim/UltiSnips/" "$HOME/.vim/"

# download vimplug
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
    
# rsync the vimwiki files from the remote server
rsync -a "remoteusr@remoteaddress" "$HOME"

vim -c PlugInstall

Other dependencies: texlive, zathura

cd /tmp # working directory of your choice
wget https://mirror.ctan.org/systems/texlive/tlnet/install-tl-unx.tar.gz
zcat < install-tl-unx.tar.gz | tar xf - # note final - on that command line
cd install-tl-2*
sudo ./install-tl --no-interaction # as root or with writable destination;
# without the explicit "perl" is also usually fine;
# omit --no-interaction if you want to select options;
# may take several hours to run.
# The path is updated by
# PATH=$PATH:/usr/local/texlive/2026/bin/x86_64-linux
# MANPATH=$MANPATH:/usr/local/texlive/2026/texmf-dist/doc/man
# INFOPATH=$INFOPATH:/usr/local/texlive/2026/texmf-dist/doc/info
# And is already written in bashprofile

sudo apt install zathura

bashrc

cp "$HOME/setup/dotfiles/bash/bashrc" "$HOME/.bashrc"
cp "$HOME/setup/dotfiles/bash/bash_aliases" "$HOME/.bash_aliases"
cp "$HOME/setup/dotfiles/bash/bash_profile" "$HOME/.bash_profile"
